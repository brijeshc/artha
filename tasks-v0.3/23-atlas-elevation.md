## Why

Assessment of the shipped dashboard (2026-07-07, on the demo): the shell is right, but the experience sits at 6.5-7/10 against the product's own bar.
Three structural gaps keep it from being a knowledge-discovery instrument:

1. **Nothing was ever drawn between things.**
   T17b mined a full import graph and flows literally span modules, yet no edge or journey was ever rendered - the single most requested reading ("architecture diagrams that link code to meaning") had no surface.
2. **The terrain is empty.**
   A module tile is a flat rectangle with a corner label; at demo scale that is hundreds of thousands of pixels carrying zero information, which reads as low quality regardless of craft elsewhere.
3. **The numbers were thin and one was a lie.**
   The top bar said "100% explained" above a queue of partly-explained modules (the exact D11 failure), and none of the computed signal (churn, coverage, freshness, history) existed as a chart.

The elevation program closes these in order: draw the linkage (23a), texture the terrain (23b), spend the numbers on charts (23c), make reading into reviewing (23d), then finish the craft debt (23e).

## Design philosophy (locked by this pivot)

**The blackboard is the organizing genre for knowledge discovery**: few marks, generous space, hand placement, chalk = light.
Every future discovery surface should ask "how would this look hand-drawn on the board?" before reaching for a denser encoding.
Charts (23c) stay in the observatory; density stays in Terrain; the board stays clean.

- [x] **State-machine layout (23e-1, shipped 2026-07-16).** Concept lifecycles are redrawn in the **chalk register** - the board's own rough strokes - so the atlas reads as one continuous hand at every altitude: repo board → module board → lifecycle.
  States are chalk boxes gone over twice, laid out **left-to-right** (the start on the left, the life advancing rightward; the longest path decides the column, so no state ever sits left of what leads into it), and a **return** (a cancel, a retry) drops into a **lane under the boxes** and runs back **orthogonally** - unmistakable, because nothing else arrives from below.
  Backed by **`stateLayout`** (`web/src/board.ts`, pure + SSR-tested): its own engine rather than an adapter over `layeredLayout`, because a lifecycle wants the states' **declared** order (read verbatim from code - that is the order the life runs in; alphabetical never is), **columns** instead of rows, and a **real cycle break** (on an import graph a cycle is an anomaly the board can seat by name, but on a state machine the returns *are* the shape - the board's fallback would collapse a fully-cyclic lifecycle into one column). Back-edges are found by DFS rooted at the true starts, so a lifecycle that loops all the way round still reads from the state the code declares first.
  The layout emits each edge's **route**, not just its endpoints, because "a way back never crosses a state it isn't about" is geometry - and geometry is testable: verticals only ever run in a column gap, the long horizontal only ever in the lane, and where a box sits directly under another (the demo's own `past_due`/`canceled`) the route steps aside into the gap first. Returns **nest** - the shortest hugs the boxes, longer ones swing out below - so they never cross each other.

- Any new hue (D2 stands everywhere: phosphor, moonlight, amber, ember, grey ink carry everything).
- Physics/force-directed layouts - the board is layered and hand-arranged, never jittering.

## Contracts produced (23a + 23a′ + 23a″ + 23b + 23c)

- Atlas route params: `lens=terrain`, `f=<flowId>`; `#/` is the board. Module route gains `?file=<path>` (23b) - the selected inner-board file, deep-linkable. New top-level route `#/observatory` (23c).
- **`GET /api/vouched-history`** → `VouchedPoint[]` (`{ at, id, kind, name }`, certified-and-dated only, oldest first) - the burn-up's raw series, read from each entry's `certified_at`, offline.
- **`GET /api/evidence?ref=<path%23Symbol>`** → `EvidenceView` (`{ ref, symbol, path, startLine, endLine, lines, truncated }`) (23d-1) - the source lines a pin was read from (D5), resolved off the repo via the cached `repoResolver`, offline and off the index; 404 when the ref no longer resolves. `EvidenceReveal`/`EvidenceCode` render it lazily under any pin.
- **`materializeInferred(repoRoot, index, id, opts)`** → `WriteOutcome` (23d-2, `src/serve/materialize.ts`) - turn an inferred concept/flow into a real `.artha/` YAML entry (`opts.certify` → certified, `opts.patch` → proposed with a correction); pure over the index, rides `commitWrite`. `POST /api/certify` and `POST /api/entry` accept an `inferred:…` id and route through it, reporting the **new human id**. Schema `BaseEntry` gains an optional **`derived_from`** (`inferred@<hash>`), the materialize provenance; the `id` suffix stays `[a-z0-9_]+`. The vouch bar is `VouchBar` in `web/src/components/Inferred.tsx`; the App navigates to the materialized page via `landAfterMaterialize`.
- **`BaseEntry.notes`** (schema + `artha_facts.notes` column + FTS body) - the delta band's human ink (D6, 23d-4), free prose the code can't hold; on `ConceptDetail`/`FlowDetail` as `notes`. Written by **`setNotes(repoRoot, id, notes)`** (`src/serve/write.ts`, additive + status-preserving like `addPin`) via **`POST /api/notes`** (`{ id, notes }`; empty clears the field). `DeltaBand` (`web/src/components/Delta.tsx`) renders it as human ink or a per-surface invitation; `ModuleDelta` is the read-only module-page framing; `Curation` gains `setNotes`. The concept states table renders empty effect/invariant cells as "not read from code" (per-field provenance).
- **`moduleReviewClaims(detail)` / `capabilityReviewClaims(detail)`** → `ReviewClaim[]` (23d-3, `web/src/derive.ts`) - the unvouched claims the review walk sweeps on a page, normalized across the moonlight (inferred) and proposed (human) tiers; certified facts, module cards, and conventions are excluded (nothing to decide). `ReviewWalk` (`web/src/components/ReviewWalk.tsx`) renders them one at a time and rides `certify` / `saveEntry` / `/api/evidence` with no new backend; `TopBar` gains `onReview` + `reviewCount` (the **Review N** pill), and the shell yields the keyboard to the walk while it is open. Reading is reviewing, page-scoped (D9).
- **`GET /api/value-queue`** → `ValueRanked[]` (23d-5, D10) - the ask queue ranked by **value not darkness**: `valueQueue` (`src/analytics/coverage.ts`, pure over churn + coverage + the reference graph) reorders the `darkZones` universe by `(1 + reach) × (1 + churn) × uncertainty`, where **`reach`** is reference in-degree (the offline agent-consumption proxy, swappable for MCP pull telemetry) and **`uncertainty` = 1 - coverage×freshness**; `ValueRanked` exposes all three factors. `whyNow(v)` (`web/src/derive.ts`) words each row's "why now" (leverage → movement → the gap, drift reported before emptiness); `QueuePage` renders the why-chips. `moduleDetail`'s `queueRank` now reads the value rank.
- `derive.flyingBlind(feed)`, `derive.areaShares(feed)`, `derive.vouchedBurnup(points)`, and `derive.standingOf(module)` - pure observatory derivations, SSR-tested; the charts (`components/Observatory.tsx`) are hand-rolled SVG like `treemap.ts`.
- `web/src/rough.ts` (seeded chalk strokes) and `web/src/board.ts` - pure, SSR-tested; every hand-drawn surface draws with them. 23b generalised the layout into **`layeredLayout(ids, links, metrics, sortKey?)`** (both `boardLayout` and `fileBoardLayout` are thin adapters over it) and extracted the drag/persist behaviour into the shared **`useBoardDrag(storeKey)`** hook.
- **`stateLayout(states, transitions)`** → `StateLayout` (23e-1, `web/src/board.ts`) - the concept lifecycle in chalk: `nodes` (left-to-right by longest path, declared order within a column), `edges` (each with `kind: 'forward' | 'return' | 'self'`, its `lane`, the orthogonal **`points`** route, and a clear `label` seat), `width`/`height`/`laneY`. Deliberately **not** an adapter over `layeredLayout` (declared order, columns, and a real cycle break are all things a lifecycle needs and an import graph does not). The route is computed here so "a return crosses no box it isn't about" is a testable invariant. `StateMachine` (`web/src/components/StateMachine.tsx`) is a dumb chalk renderer over it and takes the concept's `status` - a box wears the concept's standing, an arrow is always human ink.
- **`GET /api/module-board/:id`** → `{ module, files: [{ path, name, facts }], edges: [{ from, to }] }` (`moduleBoard`, pure over the index + `RepoStructure.files`/`fileGraph`); the inner board's data, offline.
- `derive.flowTrace(detail, modules)` and `derive.capabilitiesByModule(catalog)` - pure derivations shared by any canvas.
- **`MapModule.describedAs`** on `/api/map` - the machine's per-module prose; the 21b synthesis writes here and every reader upgrades for free.
- `Kpi.tone` gained `moon`; `AreaStat.explained` renamed to `vouched` (the honest metric everywhere).
- **`.artha/board.yaml`** + **`GET`/`POST /api/board-layout`** → `{ modules: { <module>: {x, y} } }` (23e-3, `src/serve/boardSeats.ts` `readBoardSeats`/`writeBoardSeats`) - the team's committed board. Arrangement, not meaning: never indexed, never built, invisible to MCP/export/the KPIs, and written **outside `commitWrite`** (nothing to rebuild, nothing to roll back). Sorted + rounded so an unmoved re-save is a no-op diff; empty deletes the file; a corrupt file reads as `{}` and never throws. Seat precedence is **your hand → the team's → automatic** (`useBoardDrag(storeKey, getScale?, base?)`); `BOARD.share` publishes every box's current seat and clears your local layout.
- Persisted hand layout: `localStorage['artha.board.layout.v1']` (per-browser; it layers over the committed team layout above).
