# Artha

Artha gives AI coding agents your team's **certified product-meaning** — the
`decision`s, `invariant`s, and `convention`s behind your code — so an agent
applies them mid-task **without you pasting them into the prompt**.

It mines candidate decisions from git history, lets a human certify them in a
TUI, compiles them into a queryable index, and serves them to agents over MCP
(or exports a compact `AGENTS.md` for tools that can't speak MCP).

> **Status:** v0.1, end-to-end functional. The full loop
> (`init → mine → review → build → mcp`/`export`) works; see [PROGRESS.md](PROGRESS.md).

## How it works

```
git history ──artha mine──▶ proposed drafts ──artha review──▶ certified YAML
                                                                   │
                                                              artha build
                                                                   ▼
                                                          .artha/index.db
                                                            ╱           ╲
                                                    artha mcp        artha export
                                                  (serve agents)    (AGENTS.md)
```

Nothing reaches `certified` without a human keypress — Artha drafts, **you**
certify.

## Install

Prerequisites: **Node ≥ 20** and **git**.

```bash
git clone https://github.com/brijeshc/code-artha.git
cd code-artha
npm install
npm run build        # → dist/cli.js (the `artha` bin) + dist/mcp.js (the MCP server)
```

Examples below call `artha`; until the package is published you can either
`npm link` (makes the `artha` command available globally) or substitute
`node /path/to/artha/dist/cli.js` for `artha`.

## 60-second demo

Run inside a git repository you want to capture meaning for:

```bash
artha init                      # scaffold .artha/
artha mine                      # draft decisions from git history (needs an LLM — see below)
artha review                    # certify / edit / reject the drafts in a TUI
artha build                     # compile certified YAML → .artha/index.db
artha export --agents-md        # write a compact AGENTS.md of certified entries
```

`init`, `build`, and `export` are fully offline. For example, `build` then
`export` of one certified decision:

```text
$ artha build
artha ok Built .artha/index.db — 1 entry.

$ artha export --agents-md
artha ok Wrote AGENTS.md — 1 certified entry.
```

...produces:

```markdown
<!-- Generated by `artha export --agents-md`. DO NOT EDIT — your changes will be overwritten. -->

# Product meaning (certified)

## Decisions

### decision.money — Store money as integer minor units
Represent money as integer cents everywhere; convert at the UI edge.
```

## Commands

| Command | What it does | Offline? |
|---|---|---|
| `artha init` | Scaffold `.artha/{decisions,invariants,conventions}/` and a `config.yaml`. | ✅ |
| `artha mine` | Draft `proposed` decisions from git history (heuristic pre-filter + LLM). | needs an LLM |
| `artha review` | Certify / edit / reject drafts in an interactive TUI. | ✅ |
| `artha build` | Validate + resolve pins + compile certified YAML to a SQLite/FTS5 index. | ✅ |
| `artha export --agents-md` | Emit a compact, deterministic `AGENTS.md` of certified entries. | ✅ |
| `artha mcp` | Start the stdio MCP server (`context_for_task`, `why`). | ✅ |

### `artha mine` — choosing an LLM

Mining is the one step that needs a model. Two engines, set in
`.artha/config.yaml`:

```yaml
miner:
  engine: api          # "api" (default) or "claude-cli"
  model: claude-opus-4-8
```

- **`api`** (default) — uses the Anthropic SDK. Set one of `ANTHROPIC_API_KEY`,
  `ANTHROPIC_AUTH_TOKEN`, or an `ant auth login` profile.
- **`claude-cli`** — shells out to the Claude Code CLI, reusing your existing
  Claude Code login (subscription or key) — **no separate API key needed**.

Cost is bounded by a heuristic pre-filter (merges, lockfile/format-only, and
trivial commits never reach the model) and a spend cap:

```bash
artha mine --dry-run        # preview candidate commits — zero LLM spend
artha mine --max 10         # send at most 10 commits to the miner this run
artha mine --limit 50       # only scan the 50 most-recent commits
```

Re-runs are idempotent: an already-mined commit (tracked in `.artha/.mined`) is
never drafted twice.

### `artha review` — the TUI

Runs in an interactive terminal. Each draft is shown beside its source
commit/diff and proposed pins:

- `c` certify · `e` edit (opens `$EDITOR`) · `r` reject (deletes the file) ·
  `←/→` prev/next · `q` quit

Certifying stamps `status: certified` + `certified_by` + `certified_at` and
writes valid YAML. After reviewing, run `artha build` to re-index.

## Connecting an agent (MCP)

`artha mcp` is a stdio MCP server. By default it reads `.artha/index.db` from its
working directory; if your MCP client doesn't launch it from the repo root, point
it at the repo with the `ARTHA_REPO_ROOT` env var. Use OS-native absolute paths.

For Claude Code, add it to `.mcp.json` at your project root:

```json
{
  "mcpServers": {
    "artha": {
      "command": "node",
      "args": ["C:/Code/your-repo/node_modules/artha/dist/mcp.js"],
      "env": { "ARTHA_REPO_ROOT": "C:/Code/your-repo" }
    }
  }
}
```

It exposes two tools:

- **`context_for_task(task, symbols?, files?, include_proposed?)`** — a ranked,
  token-budgeted bundle of relevant certified entries. **Certified-only by
  default**; pass the symbols/files the task touches to sharpen ranking. Stale
  entries (whose pinned code changed) are excluded.
- **`why(symbol)`** — the decision(s)/rules behind a `path#Symbol`, each tagged
  with its status.

Both are read-only and fully offline.

## Project map

```
src/
  cli.ts            commander entry → bin "artha"
  commands/         init · mine · review · build · export · mcp
  schema/           TS types + JSON Schema + AJV validator + YAML load/dump
  resolver/         tree-sitter symbol resolver + content-hash (staleness)
  build/            YAML → SQLite/FTS5 index pipeline
  mine/             heuristic pre-filter · miner engines (api / claude-cli) · ledger
  review/           Ink TUI + pure certify/reject/edit actions
  mcp/              stdio server · index read layer · ranking + token budget
  export/           AGENTS.md generator
test/               vitest suites, mirroring src/
```

| Script | Does |
|---|---|
| `npm run build` | Bundle CLI + MCP to `dist/` (tsup) |
| `npm test` | Run the vitest suite |
| `npm run typecheck` | `tsc --noEmit` |
| `npm run lint` | Lint + format check (Biome) |
