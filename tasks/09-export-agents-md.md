# Task 09 — `artha export --agents-md`

**Depends on:** 05 (read the index) — or read YAML via T02 directly; prefer reusing T08's `query.ts` read layer if available.
**Spec refs:** SPEC.md §"In scope" (`artha export --agents-md`, adoption hook), Happy path step 7.

## Goal

Emit a compact, generated `AGENTS.md` slice of **certified** entries, so flat-file-only
tools (that can't speak MCP) still get the team's certified meaning. The adoption hook.

## Scope

- **Command:** `artha export --agents-md` (T01 already routes `export`).
- **Source:** certified entries only (exclude `proposed` and `stale`). Read from `.artha/index.db` (preferred, via T08 `query.ts`) or from YAML via T02 — pick one and note it.
- **Output:** a compact, readable `AGENTS.md` grouped by kind (decisions / invariants / conventions). Per entry: heading, the one-line rule/decision, scope or pins where relevant, and `why` cross-links. Keep it terse — this is a context-budget artifact, not a dump.
- **Generated-file discipline:** clear "generated by `artha export` — do not edit" banner; deterministic output (stable ordering) so re-exports produce minimal git diffs.
- **Empty state:** no certified entries → write a valid `AGENTS.md` with a short "nothing certified yet" note (don't error).
- **Output path:** repo-root `AGENTS.md` by default; allow `--out <path>`.

## Out of scope

- Serving/ranking (T08). Export is a static snapshot, unranked.
- Including proposed/stale entries (certified-only by definition).

## Contracts produced

A generated `AGENTS.md`. No code contract consumed downstream (terminal artifact).

## Acceptance criteria  (SPEC Done-when #7)

- [ ] `artha export --agents-md` emits a compact `AGENTS.md` of certified entries, grouped by kind.
- [ ] Proposed and stale entries are excluded.
- [ ] Output is deterministic (re-running with unchanged input yields an identical file).
- [ ] Empty/no-certified state produces a valid, non-error `AGENTS.md`.
- [ ] Runs fully **offline** — SPEC Done-when #3.
